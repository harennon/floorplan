3D preview — live theme recolor

When the light/dark toggle fires while the 3D preview is active, the WebGL scene must recolor to match. LLD 129 deferred this as dead code because the theme was assumed unreachable during preview. This mockup shows the three ways the recolor could feel, and asks a prior question: should the theme toggle even be reachable during preview?

Finding while reading the code: the theme toggle (#btn-theme-toggle) sits in the topbar .actions-cluster, a sibling of .stage. No .stage--preview rule hides the topbar — so structurally the toggle is already on screen during preview; it is only unwired to a recolor path. The frontend decision is really: expose it (Options A/B) or explicitly gate it out (Option C). Toggle the theme in the header of any frame below to preview each behavior.

The whole mockup follows the same theme tokens as the real app.
Option A

Instant recolor — silent rebuild

Theme toggle stays where it is. On onThemeChange while preview.isActive(), dispose & rebuild _planGroup immediately, then render one frame. No banner, no delay — the scene just matches the new theme, exactly like re-entering preview would. Simplest, and the closest match to the LLD note ("rebuild on onThemeChange").

Watch the walls, floor and rug swap palette in place.
Option B

Recolor with a confirming cue

Same rebuild as A, but acknowledged: a brief inset flash on the canvas plus a transient "Recolored to light" chip. Because a full _planGroup dispose+rebuild can micro-stutter on a large plan, a tiny cue reassures the user the toggle registered. Costs one toast component and an animation.

Flash + chip fire on each rebuild.
Option C

Gate the toggle out of preview (keep it dead-code-free)

Honor LLD 129's original stance: while preview is active, disable the theme toggle (dimmed, aria-disabled, tooltip "Exit preview to change theme"). No recolor path ships at all — the scene re-derives colors on the next entry. This is the "nothing speculative" choice: it makes the unreachability explicit and visible rather than relying on layout accident.

Toggle is inert during preview; exit first.
