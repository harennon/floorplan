What makes a rug different (render + clearance carve-out)

  • Dashed outline, no interior glyph. Reads as a soft floor layer, not a solid object. Renders in a new #rugs SVG group painted BEFORE #symbols, so furniture always sits on top.
  • Excluded from overlap checks. computeClearances() skips rugs in BOTH subject and neighbour roles — same guard shape as CATALOG[type].openings. Dragging a sofa onto a rug never flags "bad".
  • Still selectable, movable, resizable. A rug shows the gold selection box, rotate handle, and W×H chips like any symbol — it is only the clearance role that changes, not the editing model.

Rendering note: a rug is a full-footprint fill (no type glyph) so it never competes with the furniture drawn above it. Clearance note: the carve-out is a two-line guard added to the existing symbol loop in clearance.js — if (CATALOG[sym.type]?.floorLayer) return [] (subject role) and if (CATALOG[other.type]?.floorLayer) continue (neighbour role), mirroring how openings is already excluded. Catalog entry carries a floorLayer:true flag + real presets (5×8, 8×10, 9×12 ft; runner).
