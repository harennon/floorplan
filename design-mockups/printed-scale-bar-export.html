Printed scale bar — export artifact

The exported PNG/SVG is what a user actually sends. Floorplan's wedge is to-scale measuring, so a forwarded plan needs a scale reference to be read off. This adds a labeled scale bar to buildExportSvg() in a bottom margin band — reusing EXPORT_PX_PER_M (96) and fmtLen/unitLabel.

Everything below is a live SVG rendered at EXPORT_PX_PER_M = 96, exactly as the export module would emit it. Toggle the controls to compare the three scale-bar styles, both units, both themes, and the empty-plan fallback.

How the scale bar is chosen & placed
  • Round length pick: metric snaps to 1 m (or 2 m / 5 m if the plan is very wide); imperial snaps to 1 ft / 3 ft / 10 ft. The bar's pixel width is exactly realLength × 96, so a ruler laid on the print reads true.
  • Margin band: the SVG height grows by a fixed band (≈ 0.55 m) below the existing MARGIN_M content margin, so the bar never overlaps walls, symbols, or dimension labels regardless of plan size.
  • Empty plan: renders against the default 5 × 5 m frame; the bar is still drawn (a 1 m reference) so the SVG is always valid.
  • Testable invariant: the emitted bar element's pixel length === labeled real-world length × EXPORT_PX_PER_M.
