Furniture size presets — picker directions

Each real-product size (Queen bed, 36″ fridge, Seats-6 round table) already lives on CATALOG[type].presets. Three ways to let a person pick one instead of dragging to an approximate size. Click chips / open the menus — they are live. Live footprint updates in the mini-stage.

Option A · recommended

Quick-pick chips

A wrap-around chip row added under the icon toolbar in the floating inspector. Every real size is one tap; the active size is highlighted; a dashed Custom chip appears when the user has hand-resized off-grid.

Size dropdown

One line in the inspector: a menu button showing the current named size + its dimensions. Opening reveals the full list with a checkmark on the active one. Scales cleanly for long lists (windows, doors) and stays tiny when collapsed — friendliest on mobile.

Dock long-press flyout

MCP — preset convenience arg

An agent should be able to ask for a real size by name instead of translating inches to metres. Add an optional preset:"<name>" to place_symbol that resolves to the exact {w,h} from the catalog. Explicit w/h still override; an unknown name returns a helpful error listing valid presets.

Before — agent guesses metres
place_symbol({
  type: "bed",
  x: 3.2, y: 2.1,
  // "queen is ~1.52 × 2.03?"
  w: 1.52, h: 2.03
})
After — names the product
place_symbol({
  type: "bed",
  x: 3.2, y: 2.1,
  preset: "Queen"
})
// → { ok:true, w:1.52, h:2.03,
//     preset:"Queen", clamped:false }
Unknown name — self-correcting error
place_symbol({ type:"fridge",
  x:1, y:1, preset:"Huge" })
// → { ok:false,
//   reason:"unknown preset 'Huge' for
//   fridge; try: 24\" compact, 30\",
//   33\", 36\"" }
