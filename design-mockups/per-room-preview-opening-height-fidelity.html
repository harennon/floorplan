Per-room preview scoping + opening height fidelity

LLD 132 · builds on the whole-plan preview (#101 / LLD 128 / LLD 130). Two frontend questions: (1) how a user scopes the preview to one room, and (2) how doors & windows now read as openings at the right height rather than solid slabs.

Part 1 — Per-room scoping: three entry-point options

The preview toggle (P) already exists in the tool rail. Whole-plan preview stays the default. These three options differ only in how you narrow to a single room. Pick one.

Part 2 — Opening height fidelity

Doors and windows gain sill/head height bands from the catalog. The renderer cuts the opening out of the wall at that vertical band, so a window floats in [0.9, 2.1]m and a door is a floor-to-head gap. This is data + renderer paired — no dead fields.
