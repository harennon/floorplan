Set room to W×H — affordance directions

LLD 63 follow-up. LLD 63 already made the wall dim-chip edit rectangle-preserving, so the gesture people reach for first is fixed. This explores an explicit way to type both dimensions at once for a selected rectangular room — for users who know the exact target size and don't want to edit two edges in sequence.

All three build on the existing isRectangle + rescaleRectEdge geometry and appear only when a rectangular room is selected (the dashed-gold outline from LLD 63). Weigh each against the minimal-chrome principle.

Option A W×H field docked in the Measure panel

The panel already lists the selected room's area & perimeter. When a rectangular room is selected, a compact W × H field slides in at the bottom of the panel, pre-filled with the current dimensions. Zero canvas chrome — reuses the panel users already read numbers from. Lives beside the source of truth; no floating layer.

Option B Floating W×H popover pinned to the room

On selecting a rectangular room, a small W×H card appears at the room centroid, right where the eye already is. Direct, spatially connected to the thing you're sizing. Costs a floating layer that must track pan/zoom and dismiss on deselect/Escape — more moving parts than A.

Option C Context toolbar pill above the room (dims shown, tap to edit)

A slim pill floats above the selected room showing the live W × H read-out; tapping Resize turns it into an inline W×H editor. Doubles as a passive dimension read-out and an editing entry point, and is a natural home for future room actions (delete, duplicate). Most chrome of the three.
