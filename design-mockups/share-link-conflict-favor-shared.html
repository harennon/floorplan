Share link vs. local autosave — favor the shared plan

Issue #85: opening a share link over a different autosaved plan currently shows a 50/50 chooser where local can silently win. A present hash = explicit "open this" intent and should be the safe default. Three directions below.

Toggle the states inside each stage to see the flow. Same graph-paper aesthetic and tokens as the live editor.

A — Shared-favoring banner (minimal change)

Keeps the existing top banner but rebalances it: a preview thumbnail of the incoming plan, "Open shared" as the filled primary, and "Keep my current work" demoted to a quiet ghost button. Nothing is applied until the author chooses, but the safe default is unmistakable. Smallest diff from today's code.

B — Auto-open shared + one-tap undo (recommended)

A present hash is honored immediately: the shared plan is applied and fit-to-content, exactly like the incognito / clean-open path. A non-blocking toast confirms and offers "Keep my last plan instead" for a few seconds — a one-tap undo that restores the local autosave. Zero friction for the roommate story, and the author's link never appears broken. The autosaved plan is not overwritten until the author edits, so undo is lossless.

C — Modal chooser with side-by-side previews

A centered, hard-to-miss dialog that blocks the canvas until the author decides. Both plans get a thumbnail so the choice is concrete; the shared plan is pre-selected and tagged "From this link" as the recommended default. Highest clarity, but also the heaviest interruption — best if we expect real ambiguity about which plan the author wants.
