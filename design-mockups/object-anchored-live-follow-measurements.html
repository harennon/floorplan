Measure tool · Phase 3 · Issue #93

Object-anchored measurements that live-follow

A measurement endpoint can bind to a furniture object instead of a fixed spot on the canvas. When you move, rotate, or resize that object, the anchored endpoint — and its distance label — follow it live, the way a real CAD dimension does. This mockup shows the placement affordance, the anchored-endpoint glyph, live-follow before/after, and the delete / duplicate behaviours. No new chrome: everything reuses the teal measure language already shipped in #92.

A

Placement — snap onto an object

measure tool active

Second click is being dragged over the sofa's corner. The teal snap ring appears (the existing snapType:"point" glyph). A hint tag says the endpoint will anchor to this object. Proposed default: snapping onto an object records an anchored endpoint; hold Alt to drop a plain fixed point instead.

B

Anchored vs. fixed — at rest

committed

Two committed measurements. The anchored endpoint (on the sofa) draws a hollow teal ring; the fixed endpoint draws the plain filled dot. The chip carries a small ⚭ link mark so an anchored dimension is legible at a glance.

C

Live-follow — move / rotate / resize

before → after, same measurement

The sofa was dragged right and rotated. The anchored endpoint stays welded to the sofa's near corner; the line re-aims and the label recomputes live (2.85 m → 1.40 m). The fixed endpoint on the wall never moved. Resolution happens at render time from corners() / aabb, so no stored coordinate goes stale.

D

Delete the anchored object

detach + undo

Proposed behaviour: deleting an object converts its anchored endpoints to fixed coordinates (frozen at last-known position) rather than deleting the measurement — less destructive, one undo restores both. Toast confirms and offers Undo.

E

Duplicate the anchored object

copy does not inherit

Proposed behaviour: the duplicate is a plain copy — it does not inherit the original's measurement anchor. The original keeps its live dimension (teal ring); the copy carries none. Prevents a silent, surprising second dimension appearing on the copy.

Endpoint glyph language

Fixed endpoint — filled teal dot (unchanged from #92).
Anchored endpoint — hollow ring + dot, welded to an object.
Snap candidate — pulsing ring while placing over an object.
Anchored chip — carries a ⚭ link mark.

Decision 1 · Placement default

Option A (recommended): snapping an endpoint onto an object anchors it by default; Alt drops a plain fixed point. Option B: snap stays fixed by default; a modifier (Shift) opts in to anchoring. A favours the "does my couch fit" intent (you usually measure to the couch); B is more conservative.

Decision 2 · On object delete

Option A (recommended): convert anchored endpoints to fixed coordinates (freeze in place) — non-destructive, keeps the number you were reading. Option B: delete every dependent measurement. Both must be a single undo. A loses no work; B matches strict CAD semantics.

Decision 3 · Anchor descriptor granularity

Option A (recommended): anchor to a corner index or center (matches the snap points the tool already resolves). Option B: add nearest-edge (foot-point) anchoring too — more expressive but the resolved point slides along the edge as the object resizes, which can surprise. Recommend shipping A first, B deferrable.
