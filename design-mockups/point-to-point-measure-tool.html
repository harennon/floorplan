Measure tool — anchoring & management

LLD 146 · point-to-point distance annotations · issue #87

The fixed-endpoint measure tool already shipped (PR #109 / LLD 92): a rail mode, two-click placement, snapping, teal dimension lines with ticks + distance chips, select & delete, and persistence into plan JSON. What's still undecided is how measurements behave when they touch objects and how a user manages a plan full of them. Three decisions below, each shown with the current visual language (gold walls / teal measurements / DM Mono).

Decision · object-to-object variant

When both clicks land on objects, what exactly gets measured?

Recommended: keep it point-to-point — the endpoint just snaps to the nearest corner/center of each object (already built). A separate "gap" mode (nearest-edge to nearest-edge, the clearance-style perpendicular) is a fast-follow, not v1. Below shows both so the difference is visible: teal solid = corner-to-corner (ship), dashed = nearest-edge gap (later).

Decision · endpoint anchoring (live-follow vs static)

Does a measurement follow the object it touches, or stay pinned in space?

Recommended: live-follow when an endpoint snaps to an object, fixed otherwise — with a per-endpoint toggle in the inspector. "How far is the couch from the window?" stays true as you drag the couch. Below: the couch was moved down 0.4 m; the anchored (⚓) endpoint tracked it, distance updated 2.10 → 2.35 m.

Schema

  • Endpoint = {x,y} or {ref:"s3",attach:"center"}
  • Resolve ref→point at render; fall back to last {x,y} if object deleted

Why

  • Matches CAD dimensions & the "does my couch fit" pitch
  • Per-end toggle keeps free points free

Cost

  • Delete/duplicate must reconcile refs
  • Bigger schema + validatePlan work
Decision · managing many measurements

How does a user find, select & delete measurements in a busy plan?

Today: click the line/chip on canvas to select, Delete to remove. That's enough for a few. Recommended: canvas-only for v1 (no new panel) — the chip is the affordance; selecting shows a small inline inspector (delete + anchor toggles). If a list is wanted later, it slots into the existing right-rail as a collapsible section, below Measure/Clearance. Both drawn below.
