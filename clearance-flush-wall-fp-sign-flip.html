Clearance — flush-to-wall verdict

LLD 53 · The bug: a symbol seated flush against a wall reads Won't fit — overlap on some walls but It fits / 0.0m on others, because two code paths compute the same gap and disagree by floating-point noise right where classify() splits at zero.

Touching the wall is the desired end state of the "seat against the wall" gesture — so all four walls must read identically. This is a status-classification change, so these mockups focus on what the verdict pill, gap chips, and panel say for a touching/flush state, plus the corner case from the prior iteration where flush overshot one wall.
