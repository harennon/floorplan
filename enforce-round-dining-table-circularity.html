LLD 84 · Round dining table

Enforce circularity — lock width == depth

A round dining table is a w×h box where diameter = w = h. Today both axes edit independently, so a "round" table can be dragged into an oval. These options collapse the two dimension chips into a single diameter control. Toggle the resize demo on each card to watch w and h move together.

Option A — Single diameter chip

Recommended

The two dim chips collapse into one chip on the top edge, prefixed ⌀. The depth chip is suppressed (same mechanism openings use to hide their depth). Editing it sets both axes at once. The lock-aspect button is disabled — a circle has no aspect to lock.

1.20 m

One handle, one truth. Any resize keeps w === h by construction — no way to author an oval. Presets (Seats 2/4/6/8) already ship square, so they map 1:1.

Option B — Mirrored dual chips

Alternative

Keep the familiar top (width) and left (depth) chips, but link them: editing either mirrors to the other so the shape stays circular. Both chips always read the same value. Preserves muscle memory from rectangular furniture at the cost of a redundant second handle.

1.20 m

resize_symbol on w mirrors to h and vice-versa. Circularity is still enforced, but two chips for one number can read as "why did the other change?"

Model: both options preserve the existing w×h box + circle render — only the chip layer and resize path change. A catalog flag circular: true (generalizes to lockSquare) drives the behavior: the render draws the inscribed circle unchanged, hit-testing stays box-based, and resizeSymbol mirrors the edited axis to the other, each independently clamped to the shared [0.60, 1.83] bounds. Parasol / patio-table are natural future candidates for the same flag.

Recommendation: Option A. One chip = one degree of freedom, which is the honest model for a circle, and the disabled lock button removes a control that never did anything meaningful here.
