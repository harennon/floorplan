Counter, Countertop & Island

Two new catalog types so the plan can model the continuous prep run and the free-standing island the NKBA work-triangle depends on. Per-axis bounds + real cabinetry presets, matching the post-#98 catalog shape. This mockup shows the dock entry, on-canvas glyphs, the preset picker, and the landing-zone clearance read the future check_layout evaluator will surface.

Kitchen dock2 new

bottom-center palette · Kitchen tab

Counter is a wide-but-shallow run (a base cabinet + top); Island is a free-standing block. Both slot into the existing Kitchen tab, drag onto the canvas like every other symbol, and seat flush to walls (counter) or float (island) via the LLD 26 snapping already in place.

Counter presets

selected symbol · pick a real run length
Counter 0.61 m deep · edits both axes
width 0.30–3.05 m depth 0.60–0.76 m
Why these: base cabinetry runs in 3-inch modules on a standard 24″ (0.61 m) countertop depth. Deep bound reaches 30″ for an overhang / eat-at lip. Width floor 0.30 m = a single filler; the 0.91 m preset is the NKBA continuous prep-run minimum.

Island presets

free-standing block
Island both axes
width 0.61–3.05 m depth 0.61–1.22 m
Why these: an island needs ≥0.91 m walkway on every side, so its depth bound is deeper than a wall counter. Presets track common island footprints from big-box kitchen planners.

Landing-zone rules

what a future check_layout can now evaluate
●Fridge handle side≥ 0.38 m

The unlock: today the skill can only advise in prose because no counter geometry exists. With a counter footprint on the canvas, the existing clearance.js AABB gap math (already used for "does it fit?") measures each landing zone directly — no new geometry engine needed, just two catalog entries feeding the same pipeline.
